第一章 C++语言简介

c++语言发展简史

c语言是c++的前身，在进一步扩充和完善c语言的基础上得到了c++语言。

程序设计语言分类：

  • 低级语言
    • 机器语言，依赖于CPU的指令系统，使用机器指令的二进制代码编写程序，能够直接被计算机识别（可直接操纵计算机的寄存器和内存）
    • 汇编语言，使用能够代表指令的助记符来编写程序，可以看作是符号化的机器语言（可直接操纵计算机的寄存器和内存）
  • 中级语言
  • 高级语言，面向用户的语言形式上接近于算术语言和自然语言，程序员编写方便。编写的程序易读且通用性强，大部分不能直接与硬件打交道，也不能直接在计算机上运行，需要系统软件的支持，如：需要编译程序及链接程序将高级语言编译链接为机器指令后才能运行。

c++语言特点

  • c语言的继承，尽量兼容c语言，既保持了c语言的简洁和高效，还可以像c语言那样进行结构化程序设计，同时增强了c语言对类型的处理
  • 加入了面向对象的特征，基于对象的程序设计，继承和多态为特点的面向对象的程序设计

相较c语言，c++语言的优点：

  1. 程序运行的稳定性来说，c++语言比c语言更安全，它支持过程化编程、面向对象编程和泛型编程。因为支持面向对象的开发方式，所以c++语言的应用领域更加广阔。
  2. c++可运行在多种平台，如：windows、类Unix、Macosx
  3. 加入了面向对象的概念

c++语言相对于c语言最根本的变化：引进了类和对象的概念

强制类型转换运算符

static_cast用于将一种数据类型转换成另一种数据类型，使用格式如下： static_cast<类型名>(表达式) 功能就是把表达式转换为类型名所指定的类型 static_cast也可以省略，下面四种写法都是正确的

oneInt2 = static_cast<int>(oneDouble);//强制类型转换
oneInt2 = int(oneDouble);//强制类型转换运算符的新形式
oneInt2 = (int)oneDouble; //强制类型转换运算符的旧有形式
oneInt2 = oneDouble; //自动类型转换

函数参数的默认值

c++语言规定，提供默认值时必须按从右向左的顺序提供，即有默认值的形参必须在形参列表的最后，如果有某个形参没有默认值，则它左侧的所有形参都不能有默认值。 调用函数时，主调函数的实参与被调函数的形参按从左至右的顺序进行匹配对应。

总结：声明函数时，从右向左；调用函数时，从左向右

#include<iostream>
using namespace std;
//为参数a，b，c分别设置了默认值11，22与33
void func(int a=11,int b=22,int c=33) 
{
    cout<<"a="<<a<<"，b="<<b<<"，c="<<c<<endl;
}
int main()
{ 
    func( ); //调用时缺少了3个实参，将使用定义中的3个参数默认值 输出结果：a=11,b=22,c=33
    func(55); //调用时缺少了后2个实参，将使用定义中的后2个参数默认值 输出结果：a=55,b=22,c=33
    func(77,99); //调用时缺少了最后1个实参，将使用定义中的最后1个参数默认值 输出结果：a=77,b=99,c=33
    func(8,88,888); //调用时实参完备，将不使用定义中的任何参数默认值 输出结果：a=8,b=88,c=888
    return 0;
}

void func(int a,int b=2,int c=3);// ✅ 声明时，从右向左，中间连续，不能间隔
func(1,22,333); // ✅ 调用函数时，给出了所有的实参
func( ); // ❌ 声明中的第一个参数a没有默认值，必须给出实参值
func(10,20); // ✅ 实参10，20 分别对应声明中的形参a、b，形参c使用默认值3
func(5, ,9); // ❌ 调用时，从左向右，且给出的实参必须是连续的

引用和函数参数的传递

引用概念

引用相当于变量的别名，变量对应某个内存的地址，如果给某个变量起了别名（引用），是不需要另外开辟内存空间的，相当于变量和引用都对应到同一个内存地址 在程序中使用 变量 或者 引用 都是可以的

c++中引用的定义格式如下： 类型名 &引用名 = 同类型的某变量名;

举例如下：

int oneInt;
int &aliasName = oneInt; // 声明引用 aliasName

#include<iostream>
using namespace std;
int main()
{
    int oneInt = 1;// 定义变量并赋值
    int &ref = oneInt; // 定义引用ref是oneInt的引用，ref等价于oneInt
    const int &refc = oneInt; //使用 const 定义常引用
    ref = 2; // 修改引用ref也即修改了oneInt
    cout<<"oneInt="<<oneInt<<”,"<<"ref="<<ref<<endl; // 输出oneInt=2,ref=2
    cout<<"refc="<<refc<<endl; // 输出refc=2
    oneInt = 3; // 修改oneInt变量也即修改了ref引用
    cout<<"ref=”<<ref<<endl; // 输出引用ref=3
    cout<<"refc=”<<refc<<endl; // 输出refc=3
    int & ref2 = ref; // ref2和ref都是oneInt的引用
    cout<<"ref2="<<ref2<<endl // 输出ref2=3
    //refc=5; // 错误，不能使用常引用对所引用的变量进行修改
    return 0;
}

引用在函数中的使用

  • 在程序中不仅能定义变量的引用，还可以将引用使用在函数在，引用既可以作为函数的参数使用，也可以作为函数的返回值使用
  • 在c++中，函数调用时参数的传递有两种方式：传值、传引用
  • 传值，实际上传递的就是对象的值，如果函数的形参不是引用，那么调用时实参传递给形参通常采用的是传值方式，即将实参的值拷贝给形参，在函数执行过程中，都是对这个拷贝进行操作的，函数执行完毕返回后，形参的值并不会拷贝回实参，也就是函数内部对形参的改变不会影响到函数外实参的值。
  • 传引用传递的是对象的首地址值。
